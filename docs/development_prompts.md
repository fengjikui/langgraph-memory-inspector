# Development Prompts

Use these prompts when opening a separate implementation conversation or
delegating a bounded task.

## Prompt: Build The Demo Agent

```text
You are helping build LangGraph Memory Inspector, a local-first developer tool
for inspecting LangGraph checkpoints and memory.

In the current repository, create a runnable LangGraph demo app that generates
useful checkpoint history for the inspector. The app should:

- Use Python, uv, LangGraph, and langgraph-checkpoint-sqlite.
- Run with `uv run python examples/relocation_policy_agent/run_demo.py`.
- Default to deterministic local responses so it works without an API key.
- Optionally use OpenAI when `--use-llm` and `OPENAI_API_KEY` are set.
- Save checkpoints to `examples/relocation_policy_agent/data/checkpoints.sqlite`.
- Simulate a memory bug where the user first lives in Shanghai, then moves to
  Hangzhou, but retrieval still uses the stale Shanghai memory.
- Print a final summary showing memory events, selected retrieval city,
  diagnostics, checkpoint DB path, checkpoint row count, and writes row count.
- Document that generated checkpoint artifacts are safe to delete.

Keep the implementation small and demo-first.
```

## Prompt: Build The First Inspector Backend

```text
You are helping build LangGraph Memory Inspector. Implement the first backend
slice that reads the SQLite checkpoint file generated by
`examples/relocation_policy_agent/run_demo.py`.

Create a FastAPI backend with endpoints:

- `GET /api/databases/current/summary`
- `GET /api/threads`
- `GET /api/threads/{thread_id}/checkpoints`
- `GET /api/threads/{thread_id}/checkpoints/{checkpoint_id}`
- `GET /api/threads/{thread_id}/diff?from=...&to=...`

Use a checkpoint adapter abstraction so Postgres can be added later. Start with
SQLite only. Do not build UI yet. Include tests against the generated demo DB or
a small test fixture. Keep storage behavior explicit and do not create
unbounded exports by default.
```
